Week 2: Node.js – Runtime & Core Concepts

📌 What is Node.js?

Node.js is a JavaScript runtime environment that allows you to run JavaScript outside the browser. It is mainly used for building backend applications and server-side logic.

Before Node.js, JavaScript was only used in browsers. But with Node.js, developers can build full backend systems using JavaScript.

🚀 Why Node.js is Powerful?

  • Single language for frontend & backend
  • Fast execution (V8 engine)
  • Asynchronous & non-blocking
  • Huge ecosystem (npm)

⚙️ How Node.js Works (Event Loop)

Node.js uses an event-driven, non-blocking architecture. This means it can handle multiple requests at the same time efficiently.

Instead of waiting for one task to complete, Node.js continues executing other tasks.

📦 Modules in Node.js

Node.js is modular. Everything is divided into modules.

  • Core Modules: fs, http, path
  • Custom Modules: Your own files
  • Third-party Modules: Installed via npm

📂 File System Module (fs)

Used to read and write files.

const fs = require("fs");

fs.writeFileSync("test.txt", "Hello Backend");

🌐 Creating a Server (http module)

const http = require("http");

const server = http.createServer((req, res) => {
  res.end("Hello from Backend");
});

server.listen(3000);

This is your first backend server.

📦 What is NPM?

NPM (Node Package Manager) is used to install libraries and tools. 

npm init -y
npm install express

🧠 Real-World Example

When you open a website, Node.js server handles the request, processes data, and sends response to the browser.

💡 Mini Task

  • Install Node.js
